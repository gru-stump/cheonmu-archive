# 천무 로컬 편집기 3단 스타일 설계

## 목표

현재 기능은 유지하면서 로컬 편집기에 명확한 시각적 계층과 반응형 레이아웃을 제공한다. 사용자는 항목 목록, 입력 폼, 실제 공개 화면 미리보기를 한 화면에서 비교할 수 있어야 한다.

## 시각 방향

- 공개 천무 기록 보관소의 어두운 기록실 분위기와 색상 토큰을 재사용한다.
- 편집 화면은 장시간 사용을 고려해 입력 영역의 대비, 줄 간격, 포커스 표시를 공개 열람 화면보다 선명하게 한다.
- 장식보다 가독성과 현재 상태 파악을 우선한다.
- 새 외부 UI 라이브러리는 추가하지 않는다.

## 데스크톱 레이아웃

화면은 상단 작업 표시줄, 3단 본문, 하단 변경 상태 표시줄로 구성한다.

1. 왼쪽 탐색 영역
   - 기록, 프로필, 문서, 갤러리 종류 탭
   - 검색 입력
   - 현재 종류의 항목 목록
   - 선택 항목과 미저장 항목을 시각적으로 구분
2. 가운데 편집 영역
   - 스키마 기반 입력 폼
   - 관련 기록, 공개 상태, 이미지 선택 등 기존 조작 유지
   - 필드 오류를 입력 바로 아래에 표시
   - 긴 본문 입력 영역은 화면 높이에 맞춰 충분히 확보
3. 오른쪽 미리보기 영역
   - 공개 사이트와 공유하는 표시 컴포넌트 사용
   - 기록 상태, 제목, 본문, 프로필, 갤러리 이미지를 즉시 반영
   - 미리보기와 편집 폼을 독립적으로 스크롤

기본 열 너비는 탐색 240px, 편집과 미리보기는 각각 남은 공간의 절반으로 한다. 전체 화면이 넓어져도 입력 줄 길이가 과도하게 길어지지 않도록 최대 너비를 둔다.

## 상단과 하단 상태 영역

- 상단에는 편집기 이름, 현재 항목, 변경 계획 확인 상태, 저장·삭제 동작을 배치한다.
- 저장 버튼은 변경이 있고 검증 및 권위 있는 변경 계획 조회가 성공했을 때만 활성화한다.
- 저장 또는 삭제 중에는 탐색과 중복 제출을 잠그고 진행 상태를 알린다.
- 하단에는 미저장 여부와 서버가 계산한 정확한 변경 파일 목록을 표시한다.
- 공개·비공개 이미지 목적지와 휴지통 이동 경로를 같은 목록에서 구분한다.

## 반응형 동작

- 1024px 이상: 목록, 편집, 미리보기 3단을 유지한다.
- 700px 이상 1023px 이하: 종류 탭과 가로 스크롤 항목 목록을 본문 상단에 배치하고, 그 아래 편집과 미리보기를 2단으로 유지한다.
- 699px 이하: 목록, 편집, 미리보기를 세로로 배치한다. 저장 동작은 화면 하단에서 접근 가능해야 한다.
- 모든 크기에서 가로 스크롤 없이 주요 입력과 버튼을 사용할 수 있어야 한다.

## 스타일 구성

- `src/editor/editor.css`를 편집기 전용 스타일 진입점으로 추가한다.
- 편집기 `main.tsx`에서 공용 `tokens.css`, `global.css`, `document.css`와 편집기 전용 CSS를 명시적으로 가져온다.
- 편집기 전용 클래스는 `editor-` 접두사를 사용해 공개 화면 스타일과 충돌하지 않게 한다.
- 색상, 간격, 경계선, 포커스 링은 가능한 한 공용 토큰을 사용한다.
- 로딩, 오류, 비활성화, 선택, 미저장 상태를 색상만이 아니라 문구·아이콘·경계선으로도 구분한다.

## 접근성과 오류 표시

- 모든 입력의 기존 label, `aria-invalid`, `aria-describedby` 연결을 보존한다.
- 키보드 포커스가 명확히 보이도록 `:focus-visible` 스타일을 제공한다.
- 작은 글자는 최소 12px, 일반 입력과 본문은 최소 14px 이상으로 유지한다.
- 오류 메시지는 충분한 대비를 사용하고 관련 입력 가까이에 둔다.
- `prefers-reduced-motion`에서는 장식성 전환 효과를 제거한다.

## 변경하지 않는 범위

- 로컬 API, 저장소, 검증, 휴지통, 공개·비공개 이미지 분리 동작은 변경하지 않는다.
- 콘텐츠 스키마와 편집 기능의 필드 구성은 변경하지 않는다.
- 공개 GitHub Pages 빌드에 편집기 CSS나 코드가 포함되지 않는 기존 격리를 유지한다.

## 검증

- 스타일 진입점이 없으면 실패하는 편집기 테스트를 먼저 추가한다.
- 데스크톱, 태블릿, 모바일에서 3단·2단·세로 배치를 브라우저 테스트로 확인한다.
- 입력, 저장, 삭제, 변경 계획, 미리보기의 기존 동작 테스트가 모두 통과해야 한다.
- 공개 프로덕션 빌드 후 `dist/`에 편집기 CSS, 편집기 API, 비공개 이미지 표식이 없는지 검사한다.
- 로컬 편집기를 직접 실행해 1440px, 768px, 390px에서 시각적으로 확인한다.
