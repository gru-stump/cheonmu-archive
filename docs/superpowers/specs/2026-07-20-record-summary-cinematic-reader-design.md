# 기록 요약·장면 재구성 분리와 소설형 헤더 설계

## 목표

기록 상세 페이지는 사건을 빠르게 파악할 수 있는 2~3문단 요약을 보여주고, `장면 재구성`은 별도 Markdown에 저장된 장편 본문을 소설처럼 연속 스크롤로 제공한다. 장편을 읽는 동안 제목과 닫기 버튼은 아래 스크롤에서 숨고 위 스크롤에서 다시 나타나 독서 영역을 넓힌다.

## 콘텐츠 구조

- `src/content/records/01-first-contact.md`의 body는 첫 조우의 짧은 공개 요약으로 교체한다.
- 장편 본문은 `src/content/scenes/first-contact.md`에 저장한다.
- 장면 파일 이름은 기록 `id`와 일치시켜 자동 연결한다.
- 장면 파일은 표시용 본문만 담고 별도 frontmatter를 요구하지 않는다.
- 장면 파일이 있는 cinematic 기록은 이를 우선 사용한다.
- 장면 파일이 없는 기존 cinematic 기록은 현재처럼 기록 body에서 짧은 장면 배열을 만든다.
- 장면 파일은 공개 정적 콘텐츠이므로 콘텐츠 검증에서 고아 파일과 잘못된 연결을 검사한다.

## 화면 동작

### 기록 상세

- 제목, 인용문, 장면 재구성 진입 버튼, 2~3문단 요약, 기록 정보가 표시된다.
- 전체 장편 문장은 상세 페이지 DOM에 포함하지 않는다.
- 첫 조우 요약은 격리 실패, 중상을 입은 무영, 임시 치료소의 첫 만남, 대원 확인 후 의식 상실까지만 다룬다.
- 낮은 체온, 측정값 이상, 마지막 호칭 같은 상세 단서는 장면 재구성에 남긴다.

### 장면 재구성

- 별도 장면 파일의 본문을 한 개의 prose scene으로 표시한다.
- 글자 크기는 15~17px, 최대 본문 폭은 42rem, 행간은 1.85로 유지한다.
- 이전/다음 버튼 없이 전체 본문을 연속 스크롤한다.
- 짧은 기존 cinematic 기록은 큰 연출 글씨와 장면별 이전/다음 방식을 유지한다.

## 자동 숨김 헤더

- 소설형 prose scene에서만 헤더를 sticky로 만든다.
- 스크롤 위치가 상단 안전 구간을 벗어난 뒤 아래 방향으로 이동하면 헤더를 위로 숨긴다.
- 위 방향 스크롤이 감지되면 헤더를 즉시 다시 표시한다.
- 최상단에서는 항상 표시한다.
- 닫기 버튼이 키보드 포커스를 받으면 헤더를 표시한다.
- 헤더가 시각적으로 숨은 동안에는 포인터 입력을 받지 않는다.
- `Escape` 닫기와 기존 포커스 복귀 동작은 유지한다.
- `prefers-reduced-motion` 환경에서는 이동 애니메이션을 제거한다.
- PC와 모바일에 같은 방향 규칙을 적용한다.

## 데이터 흐름

1. 콘텐츠 로더가 records와 scenes Markdown을 함께 읽는다.
2. 장면 파일의 basename을 record id와 연결해 `cinematicBody`에 담는다.
3. 상세 페이지는 `record.body`만 렌더링한다.
4. 장면 재구성은 `record.cinematicBody`가 있으면 장편 한 장면을 만들고, 없으면 기존 `record.body` 분할 방식을 사용한다.
5. CinematicScene은 현재 장면이 prose인지 판정해 소설 레이아웃과 헤더 스크롤 상태를 활성화한다.

## 오류와 호환성

- cinematic 장면 파일이 없어도 기존 기록은 정상 표시된다.
- 대응 기록이 없는 장면 파일은 콘텐츠 검증 오류로 처리한다.
- cinematic이 아닌 기록에 장면 파일이 연결되면 콘텐츠 검증 오류로 처리한다.
- 빈 장면 파일은 콘텐츠 검증 오류로 처리한다.
- 기존 stage 3·5·7 장면 순서와 키보드 조작은 변경하지 않는다.

## 테스트

- 로더: id가 같은 장면 파일을 해당 기록의 `cinematicBody`에 연결한다.
- 검증기: 고아·빈 파일 및 non-cinematic 연결을 거부한다.
- 상세 페이지: 요약은 보이고 장편 전용 문장은 보이지 않는다.
- 장면 재구성: 장편 첫 문장과 마지막 문장이 한 prose scene에 있다.
- 단위 UI: 아래 스크롤에서 헤더 숨김, 위 스크롤과 최상단·포커스에서 복귀한다.
- 접근성: `Escape` 닫기와 포커스 복귀를 유지한다.
- E2E: 1440px와 390px에서 장편 스크롤, 글자 크기, 헤더 방향 반응을 확인한다.
- 회귀: 기존 짧은 장면의 이전/다음 이동이 유지된다.

## 범위 밖

- 별도 소설 목록이나 소설 전용 라우트 추가
- 장면 파일용 시각 편집기 확장
- 공개 사이트 배포
- 다른 타임라인 기록의 장편 본문 작성
