# 천무 서사작가 설계

## 목적

`천무 서사작가`는 천령×무영 에피소드를 사용자와 공동 집필하는 프로젝트 전용 Codex 스킬이다. 짧은 사건 메모를 곧바로 본문으로 확대하지 않고, 필요한 질문과 장면 설계를 먼저 제시한 뒤 승인된 설계만 초안으로 작성한다.

초안 작성과 공개 사이트 변경은 분리한다. 본문 최종 승인만으로 파일을 수정하거나 배포하지 않으며, 사용자가 명시적으로 사이트 추가와 배포를 요청한 경우에만 기존 검증·배포 절차를 시작한다.

## 자료 우선순위

에이전트는 매 작업에서 다음 순서로 자료를 확인한다.

1. `천무_캐릭터_프로필.md` 원본 설정집
2. `src/content/profiles/`와 `src/content/documents/`의 공개 설정
3. `src/content/records/`의 기존 타임라인
4. 현재 요청에서 사용자가 제공한 사건 메모와 추가 지시

하위 자료가 상위 자료와 충돌하면 임의로 병합하거나 선택하지 않는다. 충돌 항목, 각 출처의 값, 서사에 미치는 영향을 보고하고 사용자의 결정을 기다린다. 예를 들어 현재 원본 설정집 안에서 무영의 신장은 기본 정보의 `185cm`와 확정 설정의 `189cm`가 충돌하므로 확정 전까지 어느 값도 새 본문의 사실로 단정하지 않는다.

## 문체 참고 자료

사용자가 제공하는 묘사 방법과 문체 자료는 다음 저장소 전용 스킬 경로에 둔다.

```text
.agents/skills/cheonmu-story-writer/references/style-samples/
```

이 폴더의 자료는 설정 또는 정사로 사용하지 않는다. 에이전트는 문장 길이와 호흡, 감각 묘사의 순서, 대사와 서술의 비율, 감정의 간접 표현, 복선 은폐, 장면 전환과 여운만 분석한다. 특정 작가의 고유 문장이나 문체를 그대로 재현하지 않고 여러 자료에서 표현 원리를 추출해 천무 고유의 문체 기준으로 변환한다.

권장 분류는 `sensory-description`, `emotional-tension`, `dialogue-rhythm`, `action-scenes`, `restrained-romance`, `mystery-and-foreshadowing`이다. 각 자료에는 좋아하는 부분, 천무에 적용할 부분, 피할 부분과 중요도를 메모할 수 있다. 사용자가 직접 쓴 자료가 아닌 외부 작품은 장문의 원문보다 필요한 짧은 예시와 분석 메모를 우선한다.

## 역할

하나의 스킬이 다음 네 역할을 순서대로 수행한다.

- **설정 관리자:** 확정·미확정·충돌 설정과 현재 관계 단계를 확인한다.
- **장면 설계자:** 시점, 장면 순서, 감정 변화, 분량과 수위를 제안한다.
- **본문 작가:** 사용자가 승인한 설계만 3,000~5,000자의 초안으로 작성한다.
- **연속성 편집자:** 설정, 말투, 호칭, 시점, 관계 속도와 미회수 요소를 검사한다.

역할별 판단은 사용자에게 보이는 산출물로 남긴다. 미확정 설정을 내부 판단만으로 확정하지 않는다.

## 공동 집필 흐름

### 1. 요청 정리

사건, 타임라인 단계, 분위기, 반드시 포함할 장면·대사와 금지 요소를 추출한다. 사용자가 정하지 않은 요소는 `미정`으로 유지한다.

### 2. 질문

본문의 방향을 바꾸는 정보가 부족할 때만 2~4개의 질문을 한 번에 하나씩 한다. 미확정 설정이 필요하면 서로 구분되는 2~3가지 선택지와 영향을 제시한다.

### 3. 장면 설계안

다음 항목을 포함한다.

- 제목 후보
- 타임라인상 위치
- 추천 시점과 선택 이유
- 분위기와 문체 농도
- 3~5개 장면의 순서
- 두 인물의 감정 시작점과 도착점
- 회수할 대사·행동
- 예상 분량과 로맨스 수위
- 새로 확정해야 할 설정

### 4. 설계 승인

사용자가 설계안을 승인하거나 수정하기 전에는 본문을 작성하지 않는다.

### 5. 초안 작성

기본 분량은 공백 포함 약 3,000~5,000자다. 기본 문체는 절제된 서정체이며, 감정의 전환점에서만 내면과 감각 묘사의 밀도를 높인다. 에피소드마다 가장 적합한 시점을 제안하고 승인받으며, 한 장면 안에서는 선택한 시점을 유지한다.

기본 로맨스 수위는 키스까지다. 성인 로맨스는 추후 사용자가 해당 에피소드에서 명시적으로 요청하고 별도 기준을 승인한 경우에만 다룬다.

### 6. 자가 검사

본문을 반복하지 않고 다음 결과만 짧게 첨부한다.

- 설정 충돌 여부
- 천령·무영의 말투와 호칭
- 시점 흔들림
- 관계 단계보다 앞선 행동
- 미회수 장면·대사
- 새로 발생한 미확정 설정

### 7. 피드백 반영

지적된 구간과 그 영향 범위만 수정한다. 사용자가 전체 재작성을 요청한 경우에만 전면 수정한다. 변경 뒤에는 동일한 자가 검사를 다시 수행한다.

### 8. 승인과 배포

최종 승인 이후에도 채팅 초안 상태를 유지한다. 사용자가 `사이트에 추가하고 배포해줘`처럼 저장과 배포를 명시해야 콘텐츠 파일 작성, 유효성 검사, 테스트, 커밋, 푸시와 GitHub Pages 확인을 진행한다.

## 문체와 인물 기준

- 천령은 나른하고 능글맞은 반존대를 사용한다. 감정을 농담이나 관찰로 우회하지만 위기에서는 말이 짧고 단호해진다.
- 무영은 짧고 명확한 존댓말을 기본으로 사용한다. 감정을 설명하기보다 행동과 약속으로 표현한다.
- 감정은 시선, 거리, 손길, 통증, 침묵과 대사로 먼저 드러낸다.
- 치료와 부상에는 사건의 결과와 인물의 비용이 따른다.
- 두 사람은 서로를 완전히 이해하거나 한 번의 대화로 핵심 갈등을 해소하지 않는다.
- 현재 관계 단계보다 앞선 친밀감은 사용자 승인 없이 추가하지 않는다.
- 다른 작가의 고유 문체를 그대로 모방하지 않는다.

## 단계적 정보 공개

천령의 정체와 능력은 설정집의 명칭을 본문에 옮기지 않고 다음 순서로 공개한다.

1. 독자가 먼저 감각적 위화감을 느낀다.
2. 서로 다른 맥락에서 변형된 단서를 반복한다.
3. 무영은 특수 체질, 기밀 실험, 중상으로 인한 착각처럼 합리적인 다른 해석을 세운다.
4. 독자는 천령이 평범한 인간이 아님을 무영보다 먼저 추측한다.
5. 무영이 일부 현상을 직접 목격한 뒤 정체의 일부만 확인한다.
6. 피의 성질, 실제 나이, 기원과 특재청에 머무는 이유는 별도의 공개 단계로 남긴다.

공개 전에는 `인외`, `독이자 약`, `집착`, `사랑`처럼 정답을 직접 명명하는 핵심어를 서술에 사용하지 않는다. 시점 인물이 모르는 설정은 서술자도 단정하지 않는다. 단서는 최소 세 번 다른 기능의 장면에서 축적하며, 공개 장면은 이전 장면의 의미를 다시 해석하게 해야 한다.

## 미확정 설정 정책

에이전트는 미확정 설정을 보수적으로 다룬다. 초안에 꼭 필요하면 선택지를 먼저 제시하고 사용자의 결정을 기다린다. 배경 소품처럼 정사에 영향을 주지 않는 일시적 묘사도 이후 설정으로 오인될 가능성이 있으면 `이번 장면 한정 묘사`임을 설계안에 표시한다.

## 오류와 중단 조건

다음 상황에서는 본문 작성을 멈추고 질문한다.

- 권위가 같은 자료끼리 설정이 충돌함
- 요청이 현재 관계 단계와 명백히 어긋남
- 핵심 사건의 원인이나 결과가 두 가지 이상으로 해석됨
- 요청한 수위가 사전에 승인된 범위를 넘음
- 필수 참조 파일을 읽을 수 없음

단순한 장면 디테일 부족은 중단 사유가 아니다. 서사 결과를 바꾸지 않는 범위의 선택지를 장면 설계안에서 제안한다.

## 검증

스킬을 실제 사용하기 전에 다음 시나리오로 검증한다.

1. 모호한 한 줄 메모를 받고 질문 없이 본문부터 쓰지 않는가.
2. 미확정 설정이 필요한 사건에서 임의로 정사를 만들지 않는가.
3. 장면 설계 승인 전에 본문을 작성하지 않는가.
4. 최종 초안 승인만으로 파일 수정이나 배포를 수행하지 않는가.
5. 천령과 무영의 말투·호칭·관계 단계를 구분하는가.
6. 원본 설정집 안의 `185cm`와 `189cm` 충돌을 발견하고 보고하는가.

첫 통합 시험은 타임라인 1단계 `첫 조우`를 3,000~5,000자의 에피소드로 확장하는 작업이다. 사건의 시작과 결말은 기존 기록을 유지하되, 질문과 설계 승인을 거쳐 구체적인 재난, 치료실 장면, 첫인상과 감정 도착점을 확정한다.

## 범위 밖

- 사용자 승인 없는 사이트 파일 수정 또는 배포
- 자동으로 정사를 확정하는 장기 기억 시스템
- 성인 로맨스 작성 규칙
- 공개 웹 편집기 안에 AI 생성 기능을 내장하는 작업
- 기존 에피소드의 일괄 재작성
