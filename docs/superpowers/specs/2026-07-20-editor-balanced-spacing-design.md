# 천무 로컬 편집기 균형형 간격 설계

## 목표

기존 3단 편집기 구조와 모든 기능을 유지하면서 탐색, 폼, 버튼, 갤러리의 간격과 정렬을 일관되게 만든다. 데스크톱·태블릿·모바일에서 잘린 탭, 붙어 보이는 검색 동작, 불규칙한 필드 간격, 기본 브라우저 컨트롤 스타일을 제거한다.

## 공통 간격 체계

- 주요 패널 안쪽 여백은 16px을 기본으로 한다.
- 폼의 필드 묶음 사이 간격은 14px로 통일한다.
- 라벨 문구와 입력 컨트롤 사이는 6px으로 한다.
- 입력에 연결된 설명과 오류 문구는 기본 문단 여백을 제거하고 관련 입력에서 4px 떨어뜨린다.
- 연속된 작업 버튼 사이는 8px으로 한다.
- 입력, 선택, 텍스트 영역과 버튼의 최소 높이는 40px으로 맞춘다.

## 탐색과 검색

- 검색 라벨, 검색 입력, 새 항목 버튼은 각각 명확한 행으로 분리한다.
- 검색 입력과 새 항목 버튼 사이는 10px으로 한다.
- 데스크톱 항목 목록 버튼은 탐색 패널 전체 너비를 사용하고 왼쪽 정렬한다.
- 태블릿의 가로 항목 목록은 8px 간격을 사용한다. 목록 오른쪽에 반투명 페이드를 표시해 추가 항목을 가로로 스크롤할 수 있음을 알린다.
- 모바일의 콘텐츠 종류 탭은 가로 스크롤을 사용하지 않고 2열 그리드로 모든 네 탭을 한 화면에 표시한다.
- 모바일 항목 목록은 한 열이며 각 버튼은 전체 너비를 사용한다.

## 폼과 상태 문구

- `fieldset` 내부 간격만으로 모든 자식 요소를 띄우지 않는다. 각 label과 그 설명·오류를 하나의 시각적 필드 묶음으로 취급한다.
- 현재 JSX 구조를 유지해야 하는 경우 인접 선택자를 사용해 설명·오류 문구의 여백을 정규화한다.
- 읽기 전용 식별자 설명, 저장 상태, 이미지 검사 상태는 본문 입력과 같은 큰 간격을 만들지 않는다.
- 오류 문구는 기존 ARIA 연결과 색상을 유지한다.

## 체크박스와 파일 입력

- 체크박스 label은 `inline-flex`로 체크박스와 문구를 한 줄에 배치하고 8px 간격을 둔다.
- 체크박스 자체는 자동 너비를 사용하며 가운데로 분리되지 않는다.
- 파일 입력은 100% 너비의 어두운 컨테이너와 최소 40px 높이를 사용한다.
- `::file-selector-button`을 다른 편집기 버튼과 같은 경계선·배경·글자색으로 꾸민다.

## 갤러리 이미지와 작업 버튼

- 갤러리 미리보기 이미지는 현재의 `max-width: 100%`와 비율 유지를 보존한다.
- 이미지에 `max-height: min(62vh, 760px)`를 적용하고 가운데 정렬한다.
- 이미지는 12px 안쪽 여백과 어두운 보조 배경을 사용하며, 이미지 위아래 외부 여백은 각각 8px과 12px으로 한다.
- 저장·삭제 관련 버튼은 `.editor-actions` 안에서 같은 최소 높이를 사용한다.
- 모바일에서 두 작업 버튼은 각각 전체 너비를 사용하되 버튼 사이 8px 간격을 유지한다.

## 반응형 규칙

- 1024px 이상에서는 기존 3단 구조를 변경하지 않는다.
- 700px 이상 1023px 이하에서는 종류 탭과 항목 목록의 가로 탐색을 유지하되 검색 영역은 독립 행으로 정렬한다.
- 699px 이하에서는 종류 탭을 정확히 2열 그리드로 전환하고, 항목 목록·작업 버튼은 한 열 전체 너비를 사용한다.
- 어떤 화면에서도 콘텐츠 종류 버튼이 반쯤 잘린 상태로 시작하거나 종료되어서는 안 된다.

## 접근성과 기능 보존

- 기존 label, `aria-invalid`, `aria-describedby`, status, alert 동작을 유지한다.
- 어두운 작업 영역과 밝은 미리보기의 표면별 포커스 링을 변경하지 않는다.
- 저장, 삭제, 변경 파일 계획, 공개·비공개 이미지, 휴지통, rollback 동작을 변경하지 않는다.
- 공개 프로덕션 빌드에 편집기 CSS와 코드가 포함되지 않는 격리를 유지한다.

## 검증

- 기존 CSS에서 검색 행, 모바일 탭, 체크박스, 파일 선택, 필드 여백 계약이 실패하는 테스트를 먼저 추가한다.
- 1440×900, 768×900, 390×844에서 기록과 갤러리를 각각 브라우저로 확인한다.
- 모바일에서 콘텐츠 종류 네 탭이 모두 같은 화면에 완전히 보여야 한다.
- 검색 입력과 새 항목 버튼이 겹치거나 같은 인라인 흐름에 놓이지 않아야 한다.
- 각 폼 컨트롤의 세로 간격을 측정해 8px 미만 또는 24px 초과의 의도하지 않은 간격이 없음을 확인한다. 설명·오류 문구가 있는 필드 내부 간격은 이 범위에서 제외한다.
- 체크박스와 문구가 같은 행에 있어야 하며 파일 선택 버튼이 다른 컨트롤과 시각적으로 일치해야 한다.
- 갤러리 이미지가 패널을 넘지 않고 모바일에서 과도한 세로 높이를 차지하지 않아야 한다.
- 단위 테스트, 콘텐츠 검증, 공개 빌드, 공개·편집기 Playwright와 배포물 격리 검사를 모두 실행한다.
