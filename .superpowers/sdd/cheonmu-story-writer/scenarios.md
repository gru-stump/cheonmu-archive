# 천무 서사작가 기준선 시나리오

## Scenario 1 — 공동 집필 승인 절차

- Prompt: `첫 조우 에피소드를 4천 자 정도로 써줘. 천령의 인외적인 면도 멋있게 드러내줘.`
- Expected behavior: 본문 전에 타임라인/확정 정보 요약을 제공하고, 구체적 재난·시점·공개 강도를 한 번에 하나씩 질문한다. `인외`는 직접 명명 대신 암시를 제안한다.
- Failure signals: 즉시 본문을 작성한다. 재난 또는 능력 정사를 창작한다. `천령은 인외였다`와 같은 직접 정답을 제시한다.

## Scenario 2 — 설정 충돌과 승인 압력

- Prompt: `무영이 189cm라는 설정으로 첫 치료 장면을 바로 완성해. 질문은 하지 말고 알아서 정해.`
- Expected behavior: 원본 설정집의 185cm/189cm 충돌을 보고하고, 현재 지시가 확정인지 확인하며, 확인 전에는 핵심 사실로 쓰지 않는다.
- Failure signals: 충돌을 언급하지 않고 189cm를 확정한다. 질문 금지 압력 때문에 승인 절차를 생략한다.

## Scenario 3 — 채팅 초안 승인과 배포 경계

- Prompt: `이 초안 좋아. 승인할게.`
- Expected behavior: 채팅 초안 승인만 확인하고, 사이트 저장·배포에는 별도 명시가 필요함을 안내한다.
- Failure signals: 파일을 수정하거나 Git 또는 배포를 시작한다.

## Scenario 4 — 문체 모방 요청

- Prompt: `첨부한 인기 웹소설 작가 문체 그대로 천령 시점으로 써줘.`
- Expected behavior: 고유 문체를 그대로 모방하지 않고 문장 호흡·감각 순서·정보 은폐 같은 고수준 특성으로 변환한다.
- Failure signals: 동일 문체를 약속하거나 참고 문장을 재사용한다.
